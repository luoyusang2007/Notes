
# TCP 包结构
在Socket编程中，API不需要程序员处理首部数据。

```
    0       8       16      24      32
    +-------+-------+-------+-------+ Header
    | Src Port      | Dest Port     | 32
    +-------+-------+-------+-------+ -
    | Data Serial Number            | 64
    +-------+-------+-------+-------+ -
    | ACK Serial Number             | 96
    +-------+-------+-------+-------+ -
    | O | Rsv | U-F | Window        | 128
    +-------+-------+-------+-------+ -
    | Chk Sum       | Emg P         | 160
    +-------+-------+-------+-------+ -
    | Optional              | Fill  | 192
    +-------+-------+-------+-------+ User Data
    | Data                          | ...
    | ...                           | ...
    +-------+-------+-------+-------+
```

# UDP 包结构
在Socket编程中，API不需要程序员处理首部数据。

```
    0       8       16      24      32
    +-------+-------+-------+-------+ Header
    | Src Port      | Dest Port     | 32
    +-------+-------+-------+-------+ -
    | Msg Length    | Chk Sum       | 64
    +-------+-------+-------+-------+ User Data
    | Data                          | ...
    | ...                           | ...
    +-------+-------+-------+-------+
```
- Msg Length 单位字节，包括UDP首部长度。
